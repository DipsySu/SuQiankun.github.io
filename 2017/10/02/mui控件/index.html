mui控件(一)

发表于 2017-10-02 | 分类于 混合开发

mui 是一款最接近原生App体验的高性能前端框架, 性能上避免了浏览器切页白屏,卡顿的转场动画,视觉上定制了很多精美漂亮类似安卓和iOS原生的基础控件;

比如原生控件库中存在的 accordion 折叠面板 , actionsheet 操作表, badges 数字角标 , button 定制按钮 , cardview 卡片视图 等系列基础控件

还有一些窗口管理, 事件管理, Ajax数据交互, 上下拉刷新,还有使用navigate.js对手机硬件功能进行调用;

这里介绍一下系统的原生UI仓库, mui的ui控件库是以iOS7为基础,补充部分Android的特有空间,并在此基础上封装常用的js函数,方便开发人员进行调用;

UI组件库:

折叠面板

操作表

actionsheet 一般是从底部弹出显示一系列可供用户选择的操作按钮； actionsheet是从popover控件基础上演变而来，实际上就是一个固定从底部弹出的popover，故DOM结构和popove类似，只是需要在含.mui-popover类的节点上增加.mui-popover-bottom、.mui-popover-action类；

数字角标

mui 使用自定义css样式创建了许多好看的数字角标

按钮

mui默认按钮为灰色，另外还提供了蓝色（blue）、绿色(green)、黄色(yellow)、红色(red)、紫色(purple)五种色系的按钮，五种色系对应五种场景，分别为primary、success、warning、danger、royal；使用.mui-btn类即可生成一个默认按钮，继续添加.mui-btn-颜色值或.mui-btn-场景可生成对应色系的按钮，例如：通过.mui-btn-blue或.mui-btn-primary均可生成蓝色按钮

普通按钮

在button节点上增加.mui-btn类，即可生成默认按钮；若需要其他颜色的按钮，则继续增加对应class即可，比如.mui-btn-blue即可变成蓝色按钮

加载按钮

mui v3.4 版新增加载中按钮样式,目前提供通过 JS API 切换 loading和reset状态,可以灵活操作适应多种场景

属性名 作用
data-loading-text loading状态下显示的文字,默认为loading
data-loading-icon loading状态显示为icon,默认是 mui-spinner 或者 `mui-spinner mui-spinnner-white
data-loading-icon-position loading状态下现实的icon的位置, 支持left/right. 默认是left
